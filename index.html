HTML

  • The head element contains information about the webpage
  • The charset element sets our character encoding.
  • the http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • The body (parent element) element represents the visibile content shown to the user
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The title element is also defined and sets the title of the webpage.
  • h1,h2,h3 elements represent the level of heading
  • p element represents a paragraph or block of text
  • ul, ol, and li elements represent unordered lists, ordered lists, and list items
  • br element creates a line of empthy spacy or a line break
  • footer element appears at the bottom of the page

CSS

  • A margin indicates how much space we want around the outisde of an element
  • A padding indicates how much space we want around the content inside an element
  • When using an external style sheet, we link the file using a link element placed in the HTML's head
  • The period preceding card designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS.
  • The box-shadow CSS property adds shadow effects around an element's frame. You can set multiple effects separated by commas. A box shadow is described by X and Y offsets relative to the element, blur and spread radius, and color.

Git

  • steps are: First type "git status" to see status. Second if not on the main branch, run "git checkout main" to switch to the main branch. Third create a new feature branch for the topic by entering "git checkout -b feature/add-code" and type "git status" again. Fourth add the starter code to the current working branch by typing "git add -A". Fifth enter "git commit -m "add starter code" , Sixth, then pull the remote main branch into the the local main branch by running "git pull origin main", now the local and remote main branches are in sync. Last type "git push origin feature/add-code" to create a remote branch in GitHub.
  • git status: checks what branch we are currently on
    • cd
  • git checkout -b branch name: creates a new branch and switches to it
  • README files are used to pass along useful information to anyone who visits the reporsitory on GitHub
  • git add -A: will add modifications in the current branch to the staging area. The staging area informs Git that changes will be added in the next commit
  • git commit -m "add starter code", make sure to add in quotation marks for the message
  • git pull origin main: the pull command to receive a branch's modifications into the local environment. The origin keyword indicates the source of the pull will be in the GitHub repo. The main keyword indicates the branch
  • git push origin feature/starter-code: used to push changes we have locally to our remote GitHub branch, as seen in GitHub, there will now be a remote feature/starter-code branch
  • git checkout main: updates the local main branch and navigates to the main branch
  • git checkout -b feature/starter-code: The feature/starter-code is the name of the new branch. We have added the prefix, feature, to indicate that it is a feature. It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch.

JavaScript

  • A variable is a named container that alllows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • The "const" keyword allows us to create variables that can't be reassigned a value. It essentially "locks down" what that variable will be, and once we declare it, we can never reassign it. When writing JavaScript, we always want to start by declaring variables using const.
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • Each item is assigned an index according to the order in which it is stored in the array, for example const shapes = ["triangle", "square", "pentagon", "circle"]; console.log(shapes[0,1,2,3]);
  • If we had an array that contained 100 or even 1,000 items, the most efficient way to do this is to use loops, of which there are several types. A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. Developers must use another statement, such as the conditional if statement that we just learned, to interrupt the control flow by executing a block of code a specific number of times.
  • The first statement determines the starting point for our loop. Since we want to start at the beginning of the array, we declare a variable x and give it a value of 0 (because the first index of an array is always 0), as shown in the following line of code, for example let x = 0
  • The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. This will allow the loop to execute one time for each item in the array. We use the length built-in array property on our shapes array to express that condition, as shown in the following line of code, for example x is less than sign shapes.length; length is used to account for all the variables in the array.
  • The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1. Now every time the code loops, x will increase by 1.
  • A for loop starts with—you guessed it—for. So far, our for loop looks like the following code: for(let x = 0; x is less than sign shapes.length; x++) { console.log(shapes[x]); }
  • At the most basic level, a function is simply a set of instructions that tells the computer how to perform a certain task.
  • A function is different from a for loop or conditional statement because functions do not automatically execute when the JavaScript file (in our case, script.js) is run. Instead, you must call a function by name in order to execute it.
